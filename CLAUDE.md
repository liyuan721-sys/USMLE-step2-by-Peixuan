## 三、扩展笔记前检查

新建 / 整合衍生笔记前，先 `glob` `完整笔记/专题笔记/**/*.md` 看是否已有同名笔记：
- **有** → 增量更新（保留原 frontmatter / 关联，追加新内容；版本号 v1 → v1.x）
- **无** → 有明确学科归属 → 新建在 `完整笔记/专题笔记/[学科]/xxx.md`；真正跨学科无单一归属 → `完整笔记/专题笔记/_衍生_xxx.md`（根目录，不另设子目录）

待整合的草稿放在 `草稿/`（按日期 + 版本号命名）；整合后在草稿头部加 `[已整合到正式笔记 YYYY-MM-DD]`，不删除草稿。

## 四、核心工作流：Claude.ai vs Claude Code 分工

### 总原则
**刷题阶段 = Claude.ai**；**章节整合 = Claude Code**。

| 阶段 | 谁负责 | 产出 | 存放位置 |
|---|---|---|---|
| 做题时遇到错题 | Claude.ai | 错题卡入库版（callout 折叠 + inline fields 沉底）| 用户手动贴到 `mistakes/uworld-mistakes_2026-05.md` 当日 `##` 节 |
| 错题暴露新知识缺口 | Claude.ai | 主笔记草稿（版本递增 v1→v2→v3）| `草稿/` |
| 章节复习完 | **Claude Code** | 正式主笔记 | `完整笔记/专题笔记/[学科]/xxx.md`；真正跨学科无单一归属 → `完整笔记/专题笔记/_衍生_xxx.md`（根目录）|

### Claude Code 收到"整合 XX 章节主笔记"指令时的执行步骤

```
1. 读 CLAUDE.md（本文件）
2. 找对应草稿（在 草稿/ 下，找 vX 最新版）
3. 读 mistakes/uworld-mistakes_2026-05.md 里所有相关 Q##### 错题卡
   - 可用 grep "学科:: 目标学科" 或 grep "主题:: 关键词" 检索
4. 检查 完整笔记/专题笔记/**/ 下是否已有同名笔记（按学科子目录搜全）
   - 有 → 增量更新（保留原结构，添加新内容）
   - 无 → 有明确学科归属 → 完整笔记/专题笔记/[学科]/xxx.md；真正跨学科无单一归属 → 完整笔记/专题笔记/_衍生_xxx.md（根目录）
5. 按本文件 § 五的"笔记规则"整合输出
6. 关联部分按 § 六的"关联规则 v2"处理
7. 整合完后：
   - ✅ 在草稿文件开头加 "[已整合到正式笔记 YYYY-MM-DD]"
   - ✅ 输出整合摘要给用户
   - ❌ 不删除草稿（用户决定何时归档）
   - ❌ 不修改错题本里的卡片
```

### Claude Code 绝对禁止
- ❌ **直接修改 `完整笔记/Peixuan分科笔记/` 下的学科主笔记**（除非用户明确"在学科主笔记里加 XX"）
- ❌ 修改错题本里已有的错题卡（只允许 append 新卡到末尾）
- ❌ 用主笔记替代错题本（错题在错题本，主笔记是知识图谱，分开维护）
- ❌ **修改 `错题本.md` 里的 Dataview query**（除非用户明确指令）

## 五、笔记规则

### 双语 + 风格
- **中英双语**：英文医学术语 + 中文解释，如 "Mitral Stenosis (MS, 二尖瓣狭窄)"
- 多用 callout：`[!danger]` `[!warning]` `[!tip]` `[!success]` `[!info]` `[!question]-`
- 复习题用 `> [!question]-` **可折叠** Q&A 格式
- ❌ 不用 emoji 做内容标记（用 callout 区分严重度）
- ASCII 树**必须包在 ``` 代码块**里才能等宽对齐
- ❌ 不用 Mermaid（兼容性差）

### Frontmatter / 标签
```yaml
---
tags:
  - USMLE-Step2
  - [学科名，如 Biostats / 心血管 / 神经]
created: YYYY-MM-DD
type: 专题笔记
---
```

**禁止自动加客观标签**：`#高频` `#陷阱` `#口诀` `#buzzword` `#易混` `#机制` `#反直觉` `#药物` `#ECG`

**内容标签留给 User 手动加**：`#薄弱点` `#难记` `#考前必看`

### 格式硬规则（防破损）
- 表格**前后必须空行**
- 子标题和表格之间**必须空一行**
- ASCII 树**必须包在 ``` 代码块**里
- **Markdown 表格 cell 内禁用 `[[xxx|alias]]` 形式 wikilink** — `|` 会被 linter 当成列分隔符 → 链接断裂 + 表格列错乱。表格内只用纯 wikilink `[[mistakes/uworld-mistakes_2026-05#^Q15071]]`（Obsidian 自动显示成 `Q15071`）。段落 / callout / 列表内可继续用 `|alias` 形式
- **错题卡 callout 内的图片嵌入必须带 `>` 前缀**：`> ![[image.png]]`。裸露的 `![[image.png]]` 会**打断 callout**（Obsidian 看到非 `>` 行就结束 callout），导致折叠失效 + 后续 `>` 内容变成**独立 quote block** 永久显示。同理：任何嵌入元素（图片 / pdf / 引用文件）放进 callout 都必须保持 `>` 前缀连续
- 改完后跑 awk 自查（见工作流.md）：before/after-table 必须为空

### 版权底线
- ❌ 不复制 UWorld 题干 / 答案 / 原图原文
- ✅ 用 Q##### 索引引用
- ✅ 基于 First Aid / MTB / UpToDate / AMBOSS 重新表述
- 🔒 `.gitignore` 已忽略 `*.png` / `*.jpg` / `*.pdf`

### 输出
- 主笔记输出路径：`完整笔记/专题笔记/[学科]/xxx.md`（专题，归对应学科子目录）；真正跨学科无单一归属的衍生 → `完整笔记/专题笔记/_衍生_xxx.md`（根目录，不另设 `衍生/` 子目录）
- 文件名格式：主题用中英混合可读，**不带日期**（主笔记是"活文件"）
- 草稿文件名：`YYYY-MM-DD_主题_vN.md`（带日期 + 版本号，存到 `草稿/`）

## 六、关联规则 v2（强制）

每个主笔记 / 错题卡的"关联"部分**必须包含 4 类**（少一类不行）：

```markdown
## 🔗 关联
- 🔁 同主题错题：
  - [[mistakes/uworld-mistakes_2026-05#^Q#####]] 主题一句话
  - [[mistakes/uworld-mistakes_2026-05#^Q#####]] 主题一句话
- 📚 主笔记：[[完整笔记/Peixuan分科笔记/学科名]] / [[完整笔记/专题笔记/学科/笔记名]]
- 🏥 跨学科：[[完整笔记/Peixuan分科笔记/学科]]（关联点一句话）
- 🌱 TODO（待生成衍生）：等 X 章节复习完，整合 [具体错题主题] → 生成 [[完整笔记/专题笔记/_衍生_具体名]]
```

### 核心原则
- ✅ **同主题错题（🔁）优先级最高** — 这是错题本的天然图谱
- ✅ **主笔记粗指向（📚）** — 只链笔记本身，**不写 #章节**（除非确认存在）
- ✅ **跨学科精准点出（🏥）** — 一句话说明关联点
- ✅ **主动标 TODO（🌱）** — 驱动未来衍生笔记的产生

### ❌ 禁止
- 不要写不存在的 `#章节` 锚点
- 不要堆砌看似精准但实际虚假的链接
- 不要全部链到同一个主笔记
- 不要遗漏 "Peixuan分科笔记" 或 "专题笔记" 这一级目录

## 七、错题入库格式（callout 折叠 + 知识在前 + inline fields 沉底）⭐

> [!info] 设计哲学
> 错题卡设计原则：**复习时一打开，立刻看到知识 + 个人陷阱诊断**。
> 学习记录类（提问、关联、复盘清单）和元数据（inline fields）下沉到底部，**不挡眼**。
> 
> 三段式结构：
> 
> 1. **核心知识区（前）** — 复习时反复看的内容
> 2. **学习记录区（中）** — 偶尔回顾的对话路径
> 3. **元数据 + 行动区（后）** — Dataview 用 / 复盘升档用

### 标准模板（⭐ 新顺序）

```markdown
> [!example]- [YYYY-MM-DD] 学科 / 主题 (Q#####)
> 
> ## Stem 模式
> （30 字内精简，不复制 UW 原文）
> 
> ## 核心概念
> （1-2 句话）
> 
> ## 普适规则
> （对比表 / SOP / 决策树）
> 
> ## 易混陷阱（普适）
> （非本题特有，普适性的）
> 
> ## 我为什么错
> - 选了：X
> - 错因：[知识 / pattern / 干扰项 / 粗心 / 计算]
> - 核心陷阱：...
> 
> ## Memory Hook
> （一句话锁定 + 类比）
> 
> ---
> 
> ## 🤔 我的提问 / 卡点
> （对话学习路径，关键提问 + 学到了什么）
> 
> ## 🔗 关联（按 v2 规则 4 类齐全）
> - 🔁 同主题错题：...
> - 📚 主笔记：...
> - 🏥 跨学科：...
> - 🌱 TODO：...
> 
> ## ✅ 复盘行动
> - [ ] 1 周后重做
> - [ ] 默写 [核心公式 / Stem 模式]
> - [ ] 找 3 道类似题
> 
> ---
> 
> 学科:: Biostats
> 主题:: Reporting bias
> 状态:: 🟡
> 错因:: 知识
```

### 三段式视觉分隔
- **第一个 `---`** 之前：**核心知识区**（Stem / 核心概念 / 普适规则 / 易混陷阱 / 我为什么错 / Memory Hook）
- **第一个 `---`** 之后到第二个 `---` 之前：**学习记录区**（我的提问 / 关联 / 复盘行动）
- **第二个 `---`** 之后：**元数据**（4 个 inline fields）

### inline fields 字段值规范

| 字段 | 取值范围 | 说明 |
|---|---|---|
| **学科:: ** | Biostats / Ethics / 心血管 / 肺 / 肾 / Heme-Onc / 感染 / GI / 内分泌 / GYN / OB / 神经 / Psych / 儿科 / 风湿 / 皮肤 / 眼科 / 外科 / 骨科 | 统一中文/缩写，便于 Dataview 聚合 |
| **主题:: ** | 一句话**功能性考点名**（如 "Reporting bias" / "OR-CI 表格判读"） | 不是 stem 关键词；同主题用一致命名（这样 GROUP BY 才能识别"反复错"）|
| **状态:: ** | 🔴 反复错 / 🟡 不熟（默认）/ 🟢 已掌握 | 默认 🟡；同主题第 2+ 次错升 🔴；连续 2 次同主题做对升 🟢 |
| **错因:: ** | 知识 / pattern / 干扰项 / 粗心 / 计算 | 5 选 1，对应"我为什么错"部分的根本原因 |

### Callout 类型规则（视觉分类）⭐

错题卡 callout 类型与"状态::"联动，**复习时一眼扫**"已掌握 vs 要重做"：

| Callout | 颜色 | 适用 | 触发条件 |
|---|---|---|---|
| `> [!success]-` | **绿** ✅ | **🟢 自信做对**（反射已建立）| 状态:: 🟢 且**标题不含**"做对但..."/"不确定"/"卡点"/"排除法" |
| `> [!example]-` | 紫（默认）| 错题 + **🟡 答对但不掌握** + 🔴 反复错 | 默认。包括"做对但卡里写根本卡点 / 用排除法 / 不确定 / 不懂为什么 X 错"的题 |

#### 判断逻辑（按 [feedback_correct_card_status]）⭐
- **🟢 + 标题写"✅做对 / 1:01 / 反射建立"** → `[!success]-`（绿色，复习扫过）
- **🟡 + 标题写"做对但..." / "🟡 做对不确定" / "排除法" / "做对带卡点"** → `[!example]-`（紫色，**要重点重做**）
- **错题（默认）** → `[!example]-`（紫色）

#### ❌ 反 pattern
- ❌ 不要把"答对但有卡点"标 `[!success]-` —— 绿色代表"反射建立完全掌握"，🟡 不符
- ❌ 不要看到"做对"字样就反射 `[!success]-` —— 必须扫标题是否含"做对但.../排除法/不确定/卡点"
- ✅ **"做对≠掌握"原则**：自信做对升 🟢 + success；做对但卡点 🟡 + example
- ✅ 同主题第 2+ 次错升 🔴 时**保持 `[!example]-`**（红色不另用 callout 类型，只在 inline field 体现）

### 精简原则

## 十一、每日复盘自动滚动规则（浩斯宾曲线 + 必复 + 25 题自动抽题）⭐

> [!info] 规则触发条件
> 用户说**"生成今日复盘 / 昨日学习一句话+25题自测 / 每日复盘"**等类似指令时，Claude Code **必须**按本节规则**自动**生成复盘 .md。
> 用户**不需要**手动维护任何复习日期 — 全部基于 `mistakes/uworld-mistakes_2026-05.md` 的 `## YYYY-MM-DD` 日期节自动计算。
> **核心目标**：防止用户偷懒不打开周复盘文件 — 把必复内容**强制混入**每日做题流。

### 11.1 输出文件位置 + 命名

- 路径：`复盘/YYYY-MM-DD_昨日学习_浩斯宾滚动.md`
- 日期：`today`（执行当天，不是被复盘的"昨日"）
- frontmatter：

```yaml
---
tags:
  - USMLE-Step2
  - 自测
  - 复盘
  - 浩斯宾滚动
created: YYYY-MM-DD
type: 每日复盘
---
```

### 11.2 复盘文件标准结构（精简 2 节）

```
§零 浩斯宾时间点速查（今天滚动哪些日期节）
§一 一句话总结（强制复习清单 — 昨日 + 滚动 + 必复 全部覆盖）⭐
  ├─ 1.1 D+1 昨日新学（每道错题 1 行一句话 + 一句话延伸知识点）
  ├─ 1.2 滚动复习（D+3 / D+7 / D+14 / D+30 每节抽 1 道，1 行一句话提醒）
  └─ 1.3 必复（🔴 全部 + #考前必看 全部 + #难记 轮换，每项 1 行一句话）
§二 自测 25 题（按 §11.4 比例混合抽题，盲做用 — 不含昨日 D+1，主批次为前天 D+2）
```

**§一 设计目标**：用户**光扫一句话总结**就被动复习了**所有该复习的内容** — 不打开任何笔记，纯靠每天扫这一节就完成最低限度的"接触"。**抗偷懒第一道防线**。

### 11.3 浩斯宾时间点滚动表（自动计算日期）

| 时间点 | 含义 | 日期源 |
|---|---|---|
| **D+1** | 昨日新学（**仅 §一 一句话总结，§二 不出题** — 昨日内容还记得）| `mistakes/uworld-mistakes_2026-05.md` 的 `## (today - 1 天)` 节 |
| **D+2** | 前天新学（**§二 主批次** — 防 2 天遗忘）| `## (today - 2 天)` 节 |
| **D+3** | 3 天前 | `## (today - 3 天)` 节 |
| **D+7** | 1 周前（"1 周重做"窗口）| `## (today - 7 天)` 节 |
| **D+14** | 2 周前 | `## (today - 14 天)` 节 |
| **D+30** | 1 个月前（长期巩固）| `## (today - 30 天)` 节 |

**算法**：grep `^## YYYY-MM-DD$` 找日期节 → 抽取该节内所有错题卡（`> [!example]- ...` 起头）→ 提取主题 + 一句话核心知识。

### 11.4 自测 25 题抽题比例（必须严格遵守）

> [!warning] 昨日 D+1 不进 §二
> §二 25 题**全部来自前天（D+2）及更早**。昨日（D+1）新学的内容还记得清楚，**不出题** —— 只在 §一 1.1 做一句话被动总结。每天的新学内容：day+1 进 §一 一句话（被动），day+2 进 §二 主批次（主动检测）。

| 来源 | 题数 | 抽题策略 |
|---|---|---|
| **D+2 前天新学** | **15 道** ⭐ | 前天 `##` 节内每道错题至少 1 题；不够 15 道时**从一句话总结相关知识点**出新题（生成型题）|
| **D+3** | 1 道 | 3 天前 `##` 节抽题；缺位时弹性补位 |
| **D+7** | 1 道 | 1 周前 `##` 节抽题 |
| **D+14** | 1 道 | 2 周前 `##` 节抽题 |
| **D+30** | 1 道 | 1 月前 `##` 节抽题 |
| **必复** | **4 道** ⭐ | 见 §11.5 必复抽题规则 |
| **弹性补位** | 2 道 | 当上述时间点缺题时填补；优先抽**最近 30 天内**的随机历史日期节 |
| **总计** | **25 道** | |

**前天（D+2）内容占比**：**15/25 = 60%**（核心目标 — 前天内容刚开始遗忘，趁此节点主动检测）
**滚动+必复占比**：4 + 4 = **8/25 = 32%**（防遗忘 + 防偷懒）
**弹性**：2/25 = 8%

### 11.5 必复抽题规则（4 道分配）

按**优先级动态分配** — 数量不够时按下列优先级填满 4 道：

| 优先级 | 来源 | 抽题数 | 规则 |
|---|---|---|---|
| 1 | **状态:: 🔴 反复错** 错题 | **≤2 道** | `grep "状态:: 🔴" mistakes/uworld-mistakes_2026-05.md` → 每道至少出 1 题，最多 2 道 |
| 2 | **#考前必看** 标签笔记 | **≤1 道** | grep `#考前必看` 全 vault → 优先抽最近未轮值的 |
| 3 | **#难记** 标签笔记 | **轮换 1 道** | grep `#难记` 全 vault → 按 N 取模轮换（今天 mod N，明天 mod N+1）|
| 弹性 | 4 道内如有空缺 | 填随机 🟡 错题 | 抽最近 14 天内入库的 🟡 错题 |

### 11.7 §四自测题格式要求

- 每题用 `> [!question]-` **可折叠** callout（CLAUDE.md §八规则）
- **题目下标记来源**：如 `*抽自 D+7 (2026-05-18) Q4277 — Severe AS 体征*`
- **先不给答案**（按 §八自测题规则）— 等用户作答后再讲解
- 题号 Q1-Q25，按 D+2（Q1-15）→ D+3（Q16）→ D+7（Q17）→ D+14（Q18）→ D+30（Q19）→ 必复（Q20-23）→ 弹性（Q24-25）顺序排列
- **§一 / §二 双轨制设计**：§一一句话扫一遍 = 被动复习（覆盖**昨日 D+1** + 滚动 + 必复）；§二 25 题做完 = 主动检测（覆盖**前天 D+2** + 滚动 + 必复）。昨日内容当天只在 §一 被动过一遍，到次日变 D+2 时才进 §二 主动检测。

### 11.8 缺题处理

- **某滚动时间点无错题**（如 D+3 那天没做题）→ 弹性补位区接管，不留空
- **D+2 前天没做题**（主批次 15 道无来源）→ 名额转给弹性补位 + 必复扩容，并抽最近 7 天内（D+3~D+9）历史日期节补足；**不回退到 D+1 出题**
- **必复区为空**（无 🔴 / `#考前必看` / `#难记`）→ 弹性补位区接管
- **完全空 vault**（首日）→ 出 25 道一般 USMLE 高频题（学科按弱项分布：心内/肺/肾/Heme-Onc/感染/儿科 各 3-4 道）

### 11.9 与 §八自测题规则的关系

§八规则适用："自测题先不给答案"。本节继承该规则：
- §二 25 题**只出题**，不写答案
- 用户答完 → Claude Code 判分 + 详解（参考 [[复盘/2026-05-19_5-18复盘判分_v1]] 模板）
- 判分档案文件名：`复盘/YYYY-MM-DD_浩斯宾判分_vN.md`
- §一一句话总结**直接给知识点**（被动复习用），不算自测题范围
