# 📖 周复盘 Prompt 使用说明

## 文件清单

```
完整笔记/
├── 周复盘_给ClaudeCode.md  ← 给 Claude Code 用
└── 周复盘_给ClaudeAI.md    ← 给 Claude.ai 用
```

## 30 分钟标准流程

### Step 1（5 min）：Claude Code

1. 打开 Claude Code 终端
2. 在你的 vault 路径下启动
3. 复制 `周复盘_给ClaudeCode.md` 的**全部内容**粘贴
4. 等它输出 YAML 数据
5. **复制 YAML**

### Step 2（15 min）：Claude.ai

1. 打开 Claude.ai 新对话
2. 复制 `周复盘_给ClaudeAI.md` 的**全部内容**粘贴
3. 在 `## 输入数据` 下面**贴上 Step 1 的 YAML**
4. 等深度分析
5. **复制 "给 Claude Code 的追加内容"部分**

### Step 3（10 min）：Claude Code

1. 切回 Claude Code（同一个会话或新开都行）
2. 告诉它："这是 Claude.ai 的分析结果，请执行 Step 4 追加到 薄弱点.md"
3. 粘贴 Step 2 的追加内容
4. Claude Code 会：
   - 先备份
   - 显示追加预览
   - 等你确认 yes
   - 执行追加
   - Git commit（不 push）
5. 你审核 git diff
6. 满意后 `git push`

## 关键设计原理

### 为什么分两个工具？

| 工具 | 强项 | 这次的角色 |
|------|------|-----------|
| Claude Code | 文件读写、统计、Git | **数据提取员 + 执行员** |
| Claude.ai | 长上下文、深度分析、医学知识 | **分析师 + 策略师** |

### 为什么 Claude Code 不直接分析？

- ❌ 不擅长跨科洞察
- ❌ 给的建议容易空洞（"多复习"）
- ❌ 没有医学领域优化

### 为什么 Claude.ai 不直接改文件？

- ❌ 看不到你 vault 真实内容
- ❌ 不能执行 Git
- ❌ 容易写错路径

### 为什么追加要等确认？

- ✅ 防止 Claude Code 误删 薄弱点.md 已有内容
- ✅ 你能审核追加位置
- ✅ 备份在前，回滚有底

## 故障排查

### Q: Claude Code 找不到"🔴 反复错"区域

→ Prompt 已配置：会先询问你

→ 临时方案：在 薄弱点.md 手动创建这个结构：

```markdown
# 薄弱点

## 🔴 反复错（最高优先级）

[追加内容会进入这里]

## 🟡 不熟

## 🟢 已掌握

## 📅 周复盘记录
```

### Q: 想跳过 Claude.ai，让 Claude Code 直接处理

不推荐，但可以用简化版给 Claude Code：

```
读 错题本.md
   ↓
本周 🔴 + 重做 ≥ 2 次的考点
   ↓
按学科分组追加到 薄弱点.md 的 🔴 区域（先备份，先预览，要确认）
   ↓
不做分析，不给建议
   ↓
git commit 但不 push
```

### Q: 错追加了想回滚？

```bash
cp 薄弱点_backup_YYYYMMDD.md 薄弱点.md
git checkout 薄弱点.md   # 或者用这个
```

### Q: 每次都要贴这么长的 prompt 烦不烦？

→ 可以把 `周复盘_给ClaudeAI.md` 保存为 Claude.ai 的 **Project Custom Instructions**
→ Claude Code 那边可以做成 shell alias 或 npm script

## 进阶：考前模式

考期 <2 周时，在 Claude.ai 的 prompt 末尾追加：

```
⚠️ 当前是考前 2 周模式，请用文件末尾的"考前冲刺规则"替换 Step 4
```

## 推荐放置位置

把这两个 prompt 文件放在：

```
完整笔记/周复盘_给ClaudeCode.md
完整笔记/周复盘_给ClaudeAI.md
完整笔记/周复盘_使用说明.md  ← 本文件
```

每周日打开这个文件夹就能立刻开始 ✅
