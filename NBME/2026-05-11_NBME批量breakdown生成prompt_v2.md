## 📋 给 AI 的指令（直接复制以下内容）

```
你是一位 USMLE Step 2 CK 备考助手。我会给你一份 NBME 模拟题的题目+官方解析 PDF（共 ~200 题）。

请为**每一道题**生成一个独立的 Markdown 文件，用于存入 Obsidian vault。
这些文件后续会被另一个 AI 助手（Claude Code）读取，用于跨题统计、薄弱点分析、生成个性化复习资料。

═══════════════════════════════════════════
🚨 立场声明（首要原则）
═══════════════════════════════════════════

**你不知道用户的做题结果。**

PDF 中可能出现的"错题标记"、"难题标记"、"易错警示"等都来自 PDF 准备者或官方解析，
**不代表用户本人的对错**。用户做题数据是**独立维度**，将在另一个流程中收集。

因此你必须做到：
- ❌ 不要写"这是错题/易错题"、"用户容易在这里出错"等带预判的措辞
- ❌ 不要根据"对错"调整详略——只根据**题目本身的考点难度**调整
- ❌ 不要在 frontmatter 里加 `is_wrong: true` 之类的字段
- ✅ 用中立语言：用"USMLE 考生常见思维误区"代替"你容易错"
- ✅ 难度判断只依据：考点高频度、解析复杂度、鉴别诊断密度、是否考机制 vs 单纯记忆

你的任务：为每道题生成**纯客观、与用户表现无关**的知识卡片。

═══════════════════════════════════════════

示例：

- `NBME09_Q047_Biostats_NNT计算.md`

要求：
- 套号、题号补零对齐（方便排序）
- 学科用中文（与 vault 一致）
- 主题 ≤ 10 字，能一眼看出考点
- 不出现空格、特殊字符（用下划线分隔）

​```yaml

  - 心力衰竭

                                  # 枚举：Pathophysiology / Pharmacology / Diagnosis /
                                  # Management / Prevention / Biostatistics / Ethics /
                                  # Health_Systems / Microbiology / Immunology

# === 考点元数据 ===
high_yield_concept:               # 核心考点（1-3 个，用于跨题统计）
  - PPCM 诊断三要素
  - 围产期心力衰竭时间窗

question_type:                    # 题型
  - diagnosis
  # 枚举：diagnosis / management / next_best_step / most_likely / 
  #      mechanism / risk_factor / prevention / screening / 
  #      pharmacology / biostats_calculation / biostats_interpretation /
  #      ethics_dilemma / communication / health_systems

# === 三维度陷阱标签（v2 新增）===

trap_type:                        # 题目陷阱类型（出题人埋的坑）
  - 时间窗陷阱
  # 枚举（8 类）：
  # 时间窗陷阱      —— 急性/慢性、特定时间段才适用某诊断
  # 首选vs次选       —— Best initial vs Most accurate / 一线vs二线
  # 概念混淆         —— sens/spec/PPV/NPV/RR/OR 类相似指标
  # 干扰项相似       —— 多个选项看起来都合理，要抠细节
  # 计算陷阱         —— 公式套错、单位错、分子分母颠倒
  # 机制vs症状       —— 问机制时给症状选项干扰（反之）
  # Screening vs Diagnostic —— 筛查阈值 vs 诊断阈值
  # 排除性诊断       —— 必须排除其他才能下诊断
  # 无陷阱           —— 纯知识题，无明显陷阱

thinking_error_type:              # 典型思维错误类型（这类题常引发的认知错误）
  - 锚定偏差
  # 枚举（6 类）：
  # 锚定偏差          —— 抓住第一个线索就不放，忽略后续矛盾信息
  # 模式过早闭合       —— 看到经典三联征就下诊断，没排除其他
  # 知识盲区          —— 纯粹不知道这个考点
  # 选项识别错误       —— 看错选项内容（如把 ACEI 看成 ARB）
  # 计算/操作失误      —— 公式对但算错
  # 中英文术语混淆     —— 中英对照不熟导致选错（如把 efficacy 和 effectiveness 搞反）

stem_pattern:                     # Stem 模式（题干结构特征）
  - 经典三联征
  # 枚举（8 类）：
  # 经典三联征         —— 三个特征锁定诊断（如 Cushing's triad）
  # age_sex_主诉锁定   —— 年龄+性别+主诉直接指向高发病
  # 实验室矛盾         —— 多个化验值矛盾，需识别关键值
  # 隐藏时间线         —— 时间信息分散在 stem 各处，需拼凑
  # 鉴别诊断密集       —— 给出多个相似疾病的特征，要抠细节
  # 多步推理           —— 先诊断→再选治疗→再判预后
  # 长 stem 干扰项     —— 大量无关信息测试抓重点能力
  # 简短直击           —— 一两句话直接问知识点

# === 难度与重要度 ===
difficulty: medium                # easy / medium / hard（基于题目本身，非用户对错）
high_yield_score: 4               # 1-5 分，5 = USMLE 考前必看级别
                                  # 评分依据：考点重复度、UFAP 提及频率、题型重要性

# === 关联 ===
related_vault:                    # 关联 vault 笔记（双链路径）
  - "[[完整笔记/心内]]"
  - "[[完整笔记/OB]]"
related_qid: []                   # 跨套题同主题题（初始留空，做完多套后回填）

# === 标签（严格规则）===
tags:
  - USMLE-Step2
  - 心血管                         # 仅学科分类
---
​```

三、详略自适应规则
═══════════════════════════════════════════

**判断难度仅依据题目本身**（不依据"对错"，因为你不知道用户结果）：

🟢 **easy**（estimated 30%）：纯事实回忆、单一考点、解析 < 1 页
- Part 1-4 必写
- Part 5: 压缩为 1-2 行
- Part 6: 省略
- Part 7: Memory Hook 可省，**双链必须保留**
- 目标长度：30-50 行

🟡 **medium**（estimated 50%）：需要鉴别、解析 1-2 页
- 7 部分全写，每部分最小必要长度
- 目标长度：80-120 行

🔴 **hard**（estimated 20%）：多步推理、高频考点、解析 > 2 页、易混点密集
- 7 部分全写，机制和对比表充分展开
- 必有 Memory Hook
- 可加 `> [!warning]` `> [!danger]` 提示思维误区
- 目标长度：150-250 行

**判断 high_yield_score（1-5）依据**：
- UFAP 提及度（UWorld/First Aid/Pathoma/Sketchy）
- USMLE Step 2 历年高频度
- 考点是否涉及"必考主题"（如 Sepsis 处理、ACS 流程、Biostats 计算）
- 5 = 考前必看（必有 Memory Hook）
- 4 = 高频
- 3 = 中等
- 2 = 偏门但合理
- 1 = 极冷门

## 🎯 用法 SOP（给 Peixuan 自己看）

### Phase 1: 批量预生成（每套题做一次）

1. 把上面方框内的指令喂给 Claude Code
2. 附上 NBME PDF
3. 等它输出 200 个 .md + 3 个索引文件
4. 入库到：
   ```
   NBME\nbme-records\NBME09\
   ├── _套题索引.md              ← 预生成
   ├── _学科索引.md              ← 预生成
   ├── _考点索引.md              ← 预生成
   ├── _做题结果.md              ← 你做完题后填
   └── breakdowns\               ← 200 题平铺
       ├── NBME09_Q001_xxx.md
       └── ...
   ```

### Phase 2: 做题（不动 breakdowns/）

做题时记录到 `_做题结果.md`：

```yaml
---
total_score: 168
date: 2026-05-15
duration: 2h45m
---

| QID | result | confidence | my_choice | correct | my_note |
|-----|--------|------------|-----------|---------|---------|
| 001 | right  | high       | A         | A       |         |
| 002 | wrong  | -          | B         | D       | 把 sens 和 spec 搞反 |
| 003 | right  | low        | C         | C       | 在 C/D 间纠结 |
```

字段说明：
- `result`: right / wrong
- `confidence`: high / medium / low
- `my_note`: 可选，但 wrong 和 low confidence 强烈建议填

### Phase 3: 让 Claude Code 跑复盘（下一步设计）

做完题后，让 Claude Code 读取 `_做题结果.md` + `breakdowns/*.md`，生成：
- `_错题索引.md`（按 trap_type 分类）
- `_犹豫题索引.md`（伪掌握警报）
- `_薄弱点报告.md`（结合 trap_type + thinking_error_type + stem_pattern 三维度分析）
- `_本套复习清单.md`（最该重看的 20 题）

跨套题做完几套后，再生成：
- `_跨套题高频陷阱.md`
- `_反复中招模式.md`
- `_考前 7 天必看.md`

## ⚠️ v2 关键设计澄清

1. **「中立预生成」原则**：breakdown 文件**永远是客观知识卡片**，不掺杂"对错"判断。你的做题结果是**独立维度**（在 `_做题结果.md`），通过 Claude Code 复盘时**动态结合**两者。这样最大好处是：**同一套 breakdown 适用任何用户**，未来如果你二刷或者分享给同学也能用。

2. **三维度枚举是黄金组合**：
   - `trap_type` = 题目陷阱（外部）
   - `thinking_error_type` = 思维误区（内部）
   - `stem_pattern` = 题干形态（识别层）
   - 三者交叉统计 = 真正定位你的弱点

3. **平铺 + 索引**：物理上文件平铺（Claude Code glob 一把扫），逻辑上靠索引多视角访问。这是**最 LLM 友好**的结构。
