# NBME Fighter 截图识别 Prompt

> 用这个 prompt 把 NBME Fighter 的左边栏截图转成做题结果 yaml。
> 
> **使用方式**：
> 1. 在 NBME Fighter 点开任意一题，让左边栏显示出来
> 2. 滚动截图（每屏约 25-40 题），整套 200 题大约 5-10 张截图
> 3. 把所有截图 + 当前的 `_做题结果_NBMExx.md` 一起喂给 Claude Code
> 4. 复制下方 prompt 粘贴

---

## 📋 给 Claude Code 的指令（直接复制）

```
我会给你 NBME Fighter 的题目分析页面截图（左边栏题号列表），
请精准识别每道题的状态，并更新 [[_做题结果_NBME{XX}]] 文件的 qs 数组。

═══════════════════════════════════════════
图标识别规则
═══════════════════════════════════════════

左边栏每行包含：
- 题号（数字）
- 状态图标（绿色✓ 或 红色✗）
- 可能的标记图标（红色小旗 🚩，独立存在，可与对错图标叠加）

【4 种组合】：

| 视觉 | result | mark |
|---|---|---|
| 绿色 ✓（无小旗） | right | （留空） |
| 绿色 ✓ + 红色小旗 | right | true |
| 红色 ✗（无小旗） | wrong | （留空） |
| 红色 ✗ + 红色小旗 | wrong | true |
| 无任何图标（灰色/未做） | blank | （留空） |

═══════════════════════════════════════════
识别要求（重要）
═══════════════════════════════════════════

🚨 **逐题识别，不要跳过**：
- 严格按题号顺序处理 Q001 到 Q200
- 每道题独立判断 ✓/✗ 和 🚩 两个维度
- 不要根据上下文"猜测"——只根据真实可见的图标

🚨 **截图覆盖检查**：
- 如果某题号在所有截图里都没出现 → 标注为 blank（未截到）
- 完成后给我一个 "未识别题号" 清单，让我补截图

🚨 **不要动其他字段**：
- 只更新 `result` 和 `mark` 两个字段
- 不要动 `subject`（已经从 breakdown 提取好了）
- 不要动 `my_choice`（用户做完看答案才知道选了什么，自己手填）
- 不要动 `my_note`（用户手动补充）

═══════════════════════════════════════════
输出格式
═══════════════════════════════════════════

第一步：输出识别统计

```
📊 识别结果统计
- 总题数：200
- 识别成功：xxx
- 未识别（截图未覆盖）：xxx
- 答对：xxx (其中标记 xxx)
- 答错：xxx (其中标记 xxx)
- 未答：xxx

🚨 未识别题号清单（请补截图）：Q003, Q047, Q108, ...
```

第二步：输出完整更新后的 yaml 块

```yaml
qs:
  - qid: "001"
    subject: 心内             # 保持不变
    result: right             # 你识别填的
    mark: true                # 你识别填的（无标记则留空）
    my_choice:                # 保持空
    my_note:                  # 保持空
  - qid: "002"
    subject: pulmonary
    result: right
    mark:
    my_choice:
    my_note:
  ...（直到 Q200）
```

═══════════════════════════════════════════
特别提醒
═══════════════════════════════════════════

❌ 不要做的事：
- 不要"补全"未识别的题（如果截图没覆盖到，必须标 blank，不要猜）
- 不要修改 subject 字段（如果某题 subject 为空，告诉我让我补）
- 不要给 my_choice 填值（即使你看到正确答案 D 也不要填，那是用户选项，不是答案）

✅ 必须做的事：
- 严格 200 题对齐（不能少也不能多）
- 给出未识别清单（让用户补截图）
- 截图模糊不清时，标 unsure 并列出题号

═══════════════════════════════════════════
现在请处理截图
═══════════════════════════════════════════

[在这里贴入所有截图]

当前 yaml 块（请基于此更新）：
[贴入 _做题结果_NBME12.md 文件中的 qs 数组]
```

## 💡 截图技巧（让识别更准）

### 1. 截图设置

- **缩放比例**：浏览器/软件保持 100% 缩放，不要放大或缩小
- **覆盖范围**：每张截图覆盖 25-40 题就够（题号要清晰可读）
- **重叠区域**：相邻两张截图重叠 3-5 题（防止边界识别错误）
- **完整图标**：每行的 ✓/✗ 和 🚩 都要完整不被截断

### 2. 命名建议

把截图按顺序命名：
```
NBME12_Q001-040.png
NBME12_Q041-080.png
NBME12_Q081-120.png
...
```

这样 AI 处理顺序就不会乱。

### 3. 验证流程

AI 给你 yaml 后，**抽查 5-10 道题**：
- 找你确定答对的题 → 检查 result: right
- 找你确定标记过的题 → 检查 mark: true
- 找你确定答错的题 → 检查 result: wrong

如果抽查 10 题有 0-1 题错 → AI 识别准确率 90%+，可接受
如果错 2+ → 重新提供更清晰的截图

## 🔧 进阶：批量处理多套题

如果你已经攒了 NBME 9-16 共 8 套题的截图，可以一次让 AI 处理：

```
我有 NBME 9-16 共 8 套题的截图，每套 200 题。

请按以下顺序处理：
1. NBME09 (截图 1-8 张) → 更新 [[_做题结果_NBME09]]
2. NBME10 (截图 9-16 张) → 更新 [[_做题结果_NBME10]]
...

每套处理完，给我：
- 该套统计结果
- 未识别题号清单
- 完整更新后的 yaml

然后再处理下一套。
```

注意：单次喂太多截图，AI 可能记忆混乱。**建议一套一套处理**，每套验证后再下一套。

## ❓ 常见问题

### Q1：如果 NBME Fighter 截图也包含右边的题目内容（如你给我的样图），AI 会不会误识别？

不会。Prompt 明确告诉 AI **只看左边栏的题号列表 + 图标**，不要根据右边题目内容做任何判断。

如果想更稳，截图时**裁剪掉右半部分**，只保留左边栏列表。

### Q2：如果某题的标记小旗在截图里看不太清怎么办？

让 AI 标 `unsure`，最后告诉你哪些题不确定。你打开 NBME Fighter 手动核对那几道。

### Q3：每张截图能覆盖多少题？

看你的屏幕分辨率：
- 标准笔记本 14"（1920×1080）：约 30 题/张
- 大显示器（2K/4K）：约 40-50 题/张

如果你截屏工具能滚动截图（如 Snipaste、ShareX），可以一次截全部 200 题。

### Q4：识别完后 my_choice 为什么还要我自己填？

因为 NBME Fighter 的左边栏**只显示对错状态，不显示你选了什么**。
你需要点开每道**错题/标记题**看一下自己选了什么，才能填 my_choice。

但是！其实大多数情况下你**不需要填 my_choice**——除非你想精准分析"我为什么选 B 而不是 D"。
对于 200 题里只关心 30-50 道关键题，手动补 my_choice + my_note 是合理的工作量。

## 📋 完整工作流回顾

```
做完 NBME 12 → 打开 NBME Fighter 分析页
   ↓
滚动截图左边栏（5-10 张图）
   ↓
喂给 Claude Code + 此 prompt + 当前 yaml 块
   ↓
AI 返回更新后的 yaml（含识别统计）
   ↓
贴回 _做题结果_NBME12.md
   ↓
抽查 5-10 题准确率
   ↓
打开 NBME12_学习Dashboard → 看伪掌握 / 错题
   ↓
点击关键题 → 跳到 breakdown 学习
   ↓
关键题手动补 my_choice + my_note
   ↓
打开 breakdown callout 末尾追加【我的复盘】
   ↓
跑复盘 prompt → 生成本套薄弱点报告
```

**总耗时**（200 题）：30-60 分钟
**单题平均**：10-20 秒
