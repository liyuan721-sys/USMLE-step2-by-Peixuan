# 补充专题链接 — Prompt 模板集

> 本文件包含 4 个场景的 prompt 模板，用于在 vault 演进时让 Claude Code 自动维护 breakdown 文件中的链接。
> 
> **使用方法**：找到你的场景 → 复制对应 prompt → 修改占位符 `{XXX}` → 喂给 Claude Code。

---

## 📑 场景索引

| # | 场景 | 何时用 |
|---|---|---|
| 1 | **新建专题 → 反向扫描添加链接** | 你刚手写了一个新的专题笔记 |
| 2 | **实现 🌱 TODO → 生成专题 + 双向链接** | 错题积累够了，让 AI 整合生成衍生专题 |
| 3 | **学科笔记拆分 → 升级到子页面链接** | 你把某个大学科笔记拆成多个小页面 |
| 4 | **手动指定要补的链接** | 你已经清楚要加什么链接 |

## 场景 1：新建专题 → 反向扫描添加链接

**适用情况**：
你刚新建了一个专题笔记（手写的、Claude Code 生成的、衍生的都可以），想让所有相关的 NBME 题目自动关联到这个专题。

### Prompt 模板

```
任务：为新建的专题笔记补充反向链接。

【新专题笔记】
路径：[[完整笔记/专题笔记/{专题文件名}]]
涵盖主题：{主题1}、{主题2}、{主题3}
关键词（用于匹配）：{关键词列表，逗号分隔}

例如：
路径：[[完整笔记/专题笔记/衍生/_衍生_血管性跛行鉴别]]
涵盖主题：Leriche 综合征、椎管狭窄、PAD、ABI
关键词：Leriche, claudication, aortoiliac, PAD, ABI, spinal stenosis, 跛行, 椎管狭窄

【任务】
请扫描以下文件夹下所有 NBME*_breakdown.md 文件：
NBME/nbme-records/

匹配规则（命中任一即视为相关）：
1. callout 元数据的 topic_short 包含上述关键词
2. callout 元数据的 subtopic 包含上述关键词
3. callout 元数据的 high_yield_concept 包含上述关键词

对每个匹配到的题目 callout，在「🔗 关联 vault」段落的「相关专题笔记」小节：
- 如果该小节已存在 → 追加新链接：- [[完整笔记/专题笔记/{专题文件名}]]
- 如果该小节不存在 → 创建（每行加 > 前缀，保持在 callout 内）
- 检查是否已存在该链接 → 已存在则跳过，不重复添加

【输出】
完成后给我一份「修改清单」，格式：
- NBME09_breakdown.md：在 Q012, Q047, Q089 添加了链接（共 3 题）
- NBME10_breakdown.md：在 Q003 添加了链接（共 1 题）
- 总计：N 题被更新

【严格要求】
- ❌ 不要动 callout 其他任何内容
- ❌ 不要破坏 callout 格式（每行必须 > 开头）
- ❌ 如果匹配不确定（边缘相关），列出题号让我手动确认，不要强加链接
- ✅ 修改前先汇报会改哪些题，等我确认后再动手
```

### 使用示例

假设你新建了 `_衍生_血管性跛行鉴别.md`，把上面模板里的占位符改成：

```
【新专题笔记】
路径：[[完整笔记/专题笔记/衍生/_衍生_血管性跛行鉴别]]
涵盖主题：Leriche 综合征、椎管狭窄、PAD、ABI、间歇性跛行
关键词：Leriche, aortoiliac, PAD, ABI, claudication, spinal stenosis, 跛行
```

Claude Code 跑完后会告诉你它打算在哪些题加链接，确认后才修改。

## 场景 2：实现 🌱 TODO → 生成专题 + 双向链接

**适用情况**：
breakdown 文件里很多题都有 🌱 TODO 标注（如"等血管疾病错题积累后，整合生成 [[_衍生_血管性跛行鉴别]]"）。你判断"积累够了"，让 Claude Code 真正执行这个 TODO。

### Prompt 模板

```
任务：实现 🌱 TODO，生成衍生专题笔记 + 自动建立双向链接。

【目标专题】
要生成的新专题笔记：[[完整笔记/专题笔记/{专题文件名}]]
专题主题：{专题主题}

例如：
要生成：[[完整笔记/专题笔记/衍生/_衍生_血管性跛行鉴别]]
专题主题：血管性 vs 神经源性跛行的鉴别诊断、相关疾病、ABI 应用

【步骤 1：找到所有相关 TODO】
扫描所有 NBME*_breakdown.md，找出 🌱 TODO 标注涉及该专题的所有题目。
列出清单：每道题的 NBME## + Q### + topic_short。

【步骤 2：读取这些题的核心内容】
对清单中每道题，提取：
- 一句话锁定
- 核心概念（Core Concept + 核心概念）
- 机制（ASCII 图 + 「为什么这样」）
- 对比表（如果有）
- Memory Hook

【步骤 3：生成专题笔记】
基于步骤 2 的内容，生成专题笔记，结构：

# {专题主题}

> [!info] 专题来源
> 本笔记由 N 道 NBME 题目整合生成（Q### × M）

## 📊 鉴别速查表
[把所有相关疾病的对比合并成一张大表]

## 🧠 核心机制
[整合机制内容]

## 🎯 临床决策路径
[ASCII 流程图]

## 🎣 Memory Hook
[精选最好的几个]

## 📚 涉及题目
- [[NBME09_breakdown#Q012|NBME09 Q012]] — {topic_short}
- [[NBME09_breakdown#Q047|NBME09 Q047]] — {topic_short}
- ...

## 🔗 相关学科主笔记
- （仅链接到 vault 已有的精准主题页，如果只有大文件就省略）

文件保存路径：完整笔记/专题笔记/{专题文件名}.md

【步骤 4：双向链接补全】
回到所有相关题目的 breakdown callout：
- 在「🔗 关联 vault」→「相关专题笔记」小节添加：- [[完整笔记/专题笔记/{专题文件名}]]
- 在「🌱 TODO」小节，把对应的 TODO 行**改为已完成**：
  ~~等 X 类错题积累后...~~ ✅ 已完成于 {YYYY-MM-DD}：[[完整笔记/专题笔记/{专题文件名}]]

【输出】
1. 已生成专题笔记：[[完整笔记/专题笔记/{专题文件名}]]
2. 整合题目数：N
3. 双向链接已建立的题目清单
4. 如有不确定的整合内容，列出让我审核

【严格要求】
- ❌ 不要发明医学内容（只整合现有 callout 信息）
- ❌ 不要删除原始 TODO（用删除线 + 已完成标记保留历史）
- ✅ 生成前先汇报整合计划，等我确认后再生成专题文件
```

### 使用时机

不要太早执行场景 2。建议：
- 错题/伪掌握中涉及该主题的题 **≥ 5 道**
- 你做过至少 **2-3 套** NBME
- 这样有足够素材整合，避免"专题笔记内容稀薄"

## 场景 3：学科笔记拆分 → 升级到子页面链接

**适用情况**：
你把某个大学科笔记拆分到了多个子页面（比如把"心内"拆成 PPCM、Leriche、ACS 等子文件），希望 breakdown 中的链接升级到精准的子页面。

### Prompt 模板

```
任务：将 breakdown 文件中的链接升级到学科子页面（精准跳转）。

【新建的学科子页面】
我把 {学科名} 主笔记拆分成了以下子页面：

- [[完整笔记/Peixuan分科笔记/{学科名}/{子主题1}]] — 关键词：{子主题1关键词}
- [[完整笔记/Peixuan分科笔记/{学科名}/{子主题2}]] — 关键词：{子主题2关键词}
- [[完整笔记/Peixuan分科笔记/{学科名}/{子主题3}]] — 关键词：{子主题3关键词}

例如：
- [[完整笔记/Peixuan分科笔记/心内/PPCM]] — 关键词：PPCM, peripartum, 围产期心肌病
- [[完整笔记/Peixuan分科笔记/心内/Leriche综合征]] — 关键词：Leriche, aortoiliac, 主髂闭塞
- [[完整笔记/Peixuan分科笔记/心内/ACS诊治]] — 关键词：ACS, MI, 心梗, 急性冠脉

【任务】
扫描所有 NBME*_breakdown.md：
- 找出 subject_primary = {学科名} 的所有题
- 用 topic_short / subtopic 字段匹配上述子页面关键词
- 在「🔗 关联 vault」→「相关学科精准页」小节（如不存在则新建）添加对应子页面链接

注意：v6 删除了主学科链接（因为跳到大文件顶部无意义）。
现在子页面是精准跳转 → 重新加链接是合理的。

【输出】
- 修改清单（哪些题加了哪个子页面链接）
- 未匹配上的题清单（这些题虽然是 {学科名} 但没匹配到任何子页面 → 你判断是否需要新建子页面）

【严格要求】
- ✅ 只添加子页面链接，不要再添加主学科笔记链接
- ❌ 不要写 #章节锚点
- ✅ 先汇报匹配结果，等我确认后再修改
```

### 使用注意

执行场景 3 之前，确保：
- 子页面**确实是独立小文件**（每个 < 500 行）
- 子页面的命名清晰（用主题名，不用 "笔记1" "笔记2"）
- 子页面有内容（不是空文件）

## 场景 4：手动指定要补的链接（最直接）

**适用情况**：
你已经清楚要给哪些题加什么链接，不需要 AI 做匹配判断。

### Prompt 模板

```
任务：按我提供的清单为指定题目添加链接。

【链接清单】
请在以下题目的「🔗 关联 vault」→「相关专题笔记」小节添加指定链接：

- NBME09 Q012 → [[完整笔记/专题笔记/USMLE/USMLE_临床试验分期]]
- NBME09 Q047 → [[完整笔记/专题笔记/USMLE/USMLE_高频陷阱]]
- NBME10 Q003 → [[完整笔记/专题笔记/Biostats/Biostats_6指标决策树]]
- NBME10 Q003 → [[完整笔记/专题笔记/USMLE/USMLE_广告题解题策略]]
（注：同一题可加多个链接）

【规则】
1. 找到对应 callout（按 NBME## Q### 定位）
2. 在「🔗 关联 vault」→「相关专题笔记」小节追加链接
3. 如果小节不存在 → 创建（保持每行 > 前缀）
4. 如果链接已存在 → 跳过

【输出】
完成后简单回报：成功 N 题，失败 X 题（说明原因）。
```

### 使用场景

最适合：
- 你做完一道题，复盘时想起"这题该关联到 X 专题"
- 但又懒得手动改 callout 格式
- 直接列清单交给 Claude Code

## ⚠️ 使用这些 prompt 的通用注意事项

### 1. 修改大文件前必须备份

Claude Code 在大文件上做批量修改有概率出错（删错段落、破坏 callout 格式）。

**强烈建议**：每次让 Claude Code 大批量改 breakdown 前：
1. 用 Obsidian Sync 或 git commit 保存当前状态
2. 这样出错可以一键回滚

### 2. 「先汇报，后执行」原则

上面所有模板都加了「先汇报修改计划，确认后再执行」的要求。**不要省略这一步**——
- 让 AI 列清单 → 你审核 → 说"OK 执行" → AI 修改
- 比"直接改"安全 10 倍

### 3. 每次只跑一个场景

不要同时跑场景 1 + 场景 2 + 场景 3——Claude Code 上下文混乱概率大增。
**单个 prompt 单次执行，做完检查无误再跑下一个**。

### 4. 修改后用 Obsidian 检查

每次跑完后：
1. 打开 1-2 个被修改的 breakdown 文件
2. 在 Obsidian 预览模式看 callout 是否还能正常折叠
3. 点几个新加的链接，确认能跳转
4. 如果格式坏了 → 立刻回滚到备份

## 🎯 推荐使用顺序（你的备考时间线）

### Phase 1（现在 - 做完 1-2 套 NBME）
- 不要急着补链接，先专注做题 + 复盘
- 让 🌱 TODO 自然积累

### Phase 2（做完 3-4 套 NBME）
- 跑**场景 4**：把你复盘中明确想到的关联手动批量加
- 不需要跑场景 1/2/3（积累还不够）

### Phase 3（做完 5-6 套 NBME）
- 跑**场景 2**：把 5+ 道相关题的 TODO 真正实现成专题笔记
- 跑**场景 1**：新专题反向链接到旧 breakdown

### Phase 4（考前 2-3 周）
- 跑**场景 3**：把主学科笔记拆分（如果你拆了）
- 用新建的子页面替换大文件链接（精准跳转）

### Phase 5（考前 1 周）
- 不再跑任何场景，只做题 + 复盘
- 系统已经"养成"

---

## 💡 一个意外好处

跑场景 1/2/3 的副作用：
- Claude Code 会发现你**遗漏的关联**（"我看 Q047 的内容很像 Leriche 但你没标 🌱 TODO"）
- 这些"AI 主动发现"的关联往往是你复盘时漏掉的盲点
- 让 prompt 末尾加一句"如发现意外关联请告诉我" → 收获 bonus 洞察

---

## 🆘 卡壳了？

如果某个场景跑出来效果不对（如改错了内容、破坏了格式）：
1. 立刻回滚备份
2. 把"原 prompt + AI 输出 + 期望输出"发给上游 AI（如 Claude）
3. 让它帮你调 prompt 或诊断问题

不要硬撑——单次失败的回滚成本远低于"将错就错继续改"。
