# NBME Fighter 截图识别 Prompt（v2 - 分 Section 录入版）

> **v2 更新**：
> - ✅ 用户**主动告知 section 编号**，解决"截图无 section 标识"问题
> - ✅ AI 输出**markdown 表格行**（不是 yaml），直接粘贴回方案 E 模板
> - ✅ 分 4 次提交，每次只处理 50 题（更准确、AI 不易混淆）

---

## 📋 使用流程（4 步搞定 200 题）

### Step 1：截图准备

在 NBME Fighter 里，按 4 个 section 分别截图左边栏：
- Section 1：题号 1-50
- Section 2：题号 51-100
- Section 3：题号 101-150
- Section 4：题号 151-200

每个 section 通常需要 1-2 张截图（看屏幕分辨率）。

### Step 2：第一次提交（Section 1）

把 Section 1 的截图 + 下方 prompt 喂给 Claude Code：

```
这是 NBME 12 的 Section 1 结果截图（题号 1-50）。
请按下方规则识别每道题状态，输出 markdown 表格行。

═══════════════════════════════════════════
图标识别规则
═══════════════════════════════════════════

NBME Fighter 左边栏每行包含：
- 题号（数字）
- 状态图标（绿✓ 或 红✗）
- 可能的标记图标（红色小旗 🚩，独立存在）

4 种组合 → 4 种输出：

| 看到的图标 | r/w 列 | mk 列 |
|---|---|---|
| 绿✓（无小旗） | r |   |
| 绿✓ + 红色小旗 | r | m |
| 红✗（无小旗） | w |   |
| 红✗ + 红色小旗 | w | m |
| 无任何图标 / 灰色 |   |   |

═══════════════════════════════════════════
输出格式（严格遵守）
═══════════════════════════════════════════

只输出 50 行 markdown 表格行（不要表头，不要其他文字）。
格式：| 题号 |  | r/w | mk |    |      |
注意：
- 学科列保持空（不要填，用户已经在原文件填好）
- 选/备注列保持空（用户复盘时再补）
- r/w 列只填 `r` / `w` / 留空，**不要填 right/wrong/对/错**
- mk 列只填 `m` / 留空，**不要填 marked/yes/标记**
- 数字小写字符，避免全角符号

【示例输出】
| 1  |  | r |   |    |      |
| 2  |  | r |   |    |      |
| 3  |  | w |   |    |      |
| 4  |  | r | m |    |      |
| 5  |  | w | m |    |      |
| 6  |  |   |   |    |      |   ← 未答
...
| 50 |  | r |   |    |      |

═══════════════════════════════════════════
识别要求
═══════════════════════════════════════════

🚨 严格按题号 1-50 顺序输出，**必须 50 行，不多不少**
🚨 每道题独立判断 ✓/✗ 和 🚩 两个维度
🚨 如果某题在截图里看不清，把 r/w 都留空（标记为未答），不要猜
🚨 输出完后给我一个**识别质量报告**：
   - 答对：xx 道（其中标记 xx）
   - 答错：xx 道（其中标记 xx）  
   - 未识别（截图模糊）：题号 [...]

═══════════════════════════════════════════
现在请处理 Section 1 截图
═══════════════════════════════════════════

[在这里贴 Section 1 截图]
```

### Step 3：AI 输出 → 粘贴到对应 Section 表格

AI 会输出 50 行表格数据。你打开 `_做题结果_NBME12.md`，找到 **Section 1** 区域：

```markdown
## Section 1（Item 1-50）

| Q# | 学科 | r/w | mk | 选 | 备注 |
|----|------|-----|----|----|------|
| 1  | 心内 |     |    |    |      |       ← 原本是空的
| 2  | GI   |     |    |    |      |
...
```

**关键技巧**：你不要替换整个表格（会丢掉学科列），而是用 Obsidian 的"列编辑"或者**手动只更新 r/w 和 mk 列**。

或者更简单的做法 — 让 AI 直接合并（见 Step 4 改进版 prompt）。

### Step 4：完成 Section 1 后，重复 Step 2 处理 Section 2-4

每次只发一个 section 截图，告诉 AI："这是 Section 2 结果截图（题号 51-100）"

## 💡 改进版：让 AI 直接合并表格（推荐）

如果你不想手动复制 r/w 和 mk 两列，可以改用下面这个**增强版 prompt**，AI 会读取你现有表格、保留学科列、只更新 r/w 和 mk 列：

```
这是 NBME 12 的 Section {N} 结果截图（题号 {开始}-{结束}）。

任务：
1. 读取 [[_做题结果_NBME12]] 文件中 Section {N} 的现有表格
2. 根据截图识别每道题的状态（按下方规则）
3. **保留**学科列原有内容、选列原有内容、备注列原有内容
4. **只更新** r/w 列和 mk 列
5. 输出**完整的 Section {N} 表格**（50 行，含表头）

═══════════════════════════════════════════
图标识别规则
═══════════════════════════════════════════

| 看到的图标 | r/w | mk |
|---|---|---|
| 绿✓（无小旗） | r |   |
| 绿✓ + 红旗 | r | m |
| 红✗（无小旗） | w |   |
| 红✗ + 红旗 | w | m |
| 无图标 |   |   |

【关键规则】：
- 只填小写 r/w/m，不要 right/wrong/对/错
- 学科、选、备注列保持原值（来自现有表格）
- 严格输出 50 行（不多不少）
- 表头：| Q# | 学科 | r/w | mk | 选 | 备注 |

═══════════════════════════════════════════
输出格式
═══════════════════════════════════════════

直接输出 Section {N} 完整 markdown 表格，**包含表头**：

## Section {N}（Item {开始}-{结束}）

| Q# | 学科 | r/w | mk | 选 | 备注 |
|----|------|-----|----|----|------|
| {开始} | （保留原学科） | r/w/空 | m/空 | （保留原值） | （保留原值） |
...
| {结束} | ... | ... | ... | ... | ... |

完成后输出统计：
- 答对 X 道（其中标记 X）
- 答错 X 道（其中标记 X）
- 未识别 X 道（题号：...）

═══════════════════════════════════════════
处理截图
═══════════════════════════════════════════

[贴入 Section {N} 截图]
```

你只需要把这段复制粘贴 4 次，每次改 `{N}` 和 `{开始}-{结束}`，就能处理完整 4 个 section。

## 🎯 完整工作流示例

### 假设你做完了 NBME 12，开始填表：

**第 1 轮**：
1. NBME Fighter 显示题号 1-50（Section 1）
2. 截图（1-2 张覆盖全部 50 题）
3. 复制改进版 prompt，把 `{N}` 改成 `1`，`{开始}` 改成 `1`，`{结束}` 改成 `50`
4. 发给 Claude Code + 截图
5. AI 输出完整 Section 1 表格
6. 你把 AI 输出的表格替换到 `_做题结果_NBME12.md` 中 **Section 1（Item 1-50）** 部分

**第 2 轮**：
1. NBME Fighter 滚动到题号 51-100
2. 截图
3. 改 prompt 的 `{N}=2`、`{开始}=51`、`{结束}=100`
4. 发给 Claude Code → 替换 Section 2

**第 3 轮、第 4 轮**：重复

**总耗时**：4 × 3 分钟 = **12 分钟**完成 200 题状态录入。

## 📝 提交 prompt 的极简版（推荐日常用）

如果你觉得上面 prompt 太长，每次复制改 N 麻烦，可以用这个**极简版**：

```
这是 NBME 12 Section 1 截图（题号 1-50）。

请：
1. 读取 [[_做题结果_NBME12]] 的 Section 1 表格
2. 用截图更新 r/w 和 mk 列（小写 r/w/m，看到红旗就 m）
3. 保留学科、选、备注列原值
4. 输出完整 Section 1 表格（50 行 + 表头）
5. 最后报告：对X错X标记X未识别X

[贴截图]
```

下次切到 Section 2，只需改两个数字：

```
这是 NBME 12 Section 2 截图（题号 51-100）。
（其余同上）
```

## ⚠️ 关键注意事项

### 1. NBME Fighter 不显示 section，靠你自己分

NBME Fighter 左边栏只有连续题号 1-200。所以**你要在 NBME Fighter 里手动滚动到对应区间再截图**：

| Section | 题号范围 | 截图时确认题号 |
|---|---|---|
| 1 | 1-50 | 第一张可见 Q1，最后一张可见 Q50 |
| 2 | 51-100 | 第一张可见 Q51，最后一张可见 Q100 |
| 3 | 101-150 | 第一张可见 Q101，最后一张可见 Q150 |
| 4 | 151-200 | 第一张可见 Q151，最后一张可见 Q200 |

### 2. 一次只发一个 section

不要把 4 个 section 截图一次性发——AI 容易混淆。**严格 4 次提交**，每次告诉它处理哪个 section。

### 3. 输出表格后必须保存

AI 给你的是文本输出，你**复制 → 粘贴到 obsidian 文件 → Ctrl+S 保存**。Dataview 才能读到。

### 4. 学科列必须先填好

在你用截图识别 prompt **之前**，必须先让 v4 prompt 把所有题的 subject 填到表格里（见"开始一套 NBME"的 Step 3）。否则 AI 没东西保留，会留空。

## ❓ 常见问题

### Q1：如果一个 section 需要多张截图（屏幕小）怎么办？

把所有覆盖这个 section 的截图一起发给 AI，告诉它："这是 Section 1 的全部截图，共 X 张"。AI 会自动合并识别。

### Q2：抽查发现 AI 识别错了 1-2 题怎么办？

直接手动改那 1-2 题的 r/w 或 mk 字符。Dataview 会自动重新统计。

### Q3：能不能让 AI 一次处理 200 题？

**不推荐**。原因：
- 200 题的截图大概 8-10 张，单次上下文太大
- AI 容易把不同 section 的题号搞混
- 出错后不好定位是哪一段错了

**4 次提交是最稳定的工作流**。

### Q4：截图识别准确率大概多少？

NBME Fighter 的图标对比鲜明（绿✓ vs 红✗ vs 红旗），AI 识别准确率应该 **95%+**。
如果你发现准确率明显低（多于 5%），可能是：
- 截图缩放/分辨率有问题
- 截图被压缩损失
- 部分题号被滚动条遮挡

### Q5：能识别 NBME 软件以外的题库（如 UWorld）截图吗？

理论上可以，但需要**修改图标识别规则**。UWorld 的对错图标和标记图标长得不一样。给我发一张 UWorld 截图，我能为你定制识别 prompt。
