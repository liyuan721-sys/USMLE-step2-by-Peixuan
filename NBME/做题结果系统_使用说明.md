# NBME 做题结果系统 使用指南

> 本系统由 3 个核心文件组成：
> 1. **`_做题结果模板_NBMExx.md`** — 每套 NBME 复制一份用
> 2. **`NBME_汇总Dashboard.md`** — 全局 Dashboard，自动聚合所有套题
> 3. （配合）**`NBME{xx}_breakdown.md`** — 由 v4 prompt 批量生成的题目详解

---

## 🛠️ 一次性准备（5 分钟）

### 1. 安装 Dataview 插件

Obsidian → 设置 → 第三方插件 → 浏览 → 搜索 `Dataview` → 安装 → 启用

⚠️ **必须开启** JS 查询：Dataview 设置 → "Enable JavaScript Queries" → 打开

（本模板里的统计、错题快速跳转、汇总 Dashboard 全靠 `dataviewjs` 代码块）

### 2. 文件放置

```
NBME/nbme-records/
├── NBME_汇总Dashboard.md           ← 放这里，全局唯一
├── _做题结果模板_NBMExx.md         ← 模板，每套复制一份
├── NBME09/
│   ├── _做题结果_NBME09.md         ← 从模板复制改名
│   ├── NBME09_breakdown.md         ← v4 prompt 生成
│   ├── NBME09_套题索引.md
│   ├── NBME09_学科索引.md
│   └── NBME09_考点索引.md
├── NBME10/
│   ├── _做题结果_NBME10.md
│   └── ...
└── ...
```

### 3. 测试 Dataview

打开 `NBME_汇总Dashboard.md`，应该看到：
- ✅ 表格能渲染（哪怕是空的）
- ❌ 如果只看到代码而不是表格 → JS 查询没开

## 📅 日常工作流

### 阶段 1：准备做题（做题前）

**第 1 步**：从模板复制一份

```bash
copy _做题结果模板_NBMExx.md NBME12\_做题结果_NBME12.md
```

**第 2 步**：让 Claude Code 批量初始化 200 题占位

打开新文件，找到 `qs:` 那个 YAML 块。它只有 5 题占位。让 Claude Code 帮你扩展到 200 题：

```
请把这个 yaml 块的 qs 数组扩展到 Q001-Q200，每题保留空值占位：

[贴入当前的 5 题 yaml]
```

**第 3 步**：更新 frontmatter

```yaml
---
nbme: 12              # 改成 12
date: 2026-05-15      # 计划做题日期
status: not_started
total_score: 
target: 250
duration_minutes: 
review_pass: 0
tags:
  - USMLE-Step2
  - NBME模拟题
  - 做题结果
---
```

---

### 阶段 2：做题（模拟考试环境）

⚠️ **做题时不要打开这个文件**——你选了"做完整套后一次性填"，遵循模拟考试纪律。

只用 NBME 软件自带的「Mark」功能标记不确定的题。

---

### 阶段 3：填表（做完立刻做，不要拖到第二天）

打开 `_做题结果_NBME12.md`，找到这一段：

```yaml
qs:
  - qid: "001"
    result: right        ← 在这里填
    conf: high           ← 在这里填
    my_choice: A
    my_note: 
```

**填写规则**：

| 字段 | 可填值 | 说明 |
|---|---|---|
| `result` | `right` / `wrong` / `marked` / `blank` | 答对/答错/标记/未答 |
| `conf` | `high` / `medium` / `low` | 高/中/低置信度 |
| `my_choice` | `A` / `B` / `C` / `D` / `E` / `F` | 你选的字母 |
| `my_note` | 任意中文 | 可选，但 wrong + low conf 强烈建议填 |

#### 💡 提速技巧：用 Find & Replace 批量填

填完发现大多数题都是 `result: right` + `conf: high`，每行手填太慢？

1. 先用 Ctrl+H 把所有空白 result 批量填成 `right`（默认值）
2. 然后只去改那些不是"答对+高置信度"的题（通常 50-80 题）
3. 速度从 30 分钟降到 10 分钟

#### 💡 双轨制：让 Claude Code 帮你填对错

如果你不想手敲 200 个 `right`：

1. 在 NBME 软件上截图所有题的对错统计页（通常 5-10 张能覆盖全部）
2. 把截图喂给 Claude Code：

```
请读取这些截图，识别 NBME12 每道题的对错，
然后帮我更新这个文件的 qs 数组中的 result 字段：
- 答对 → result: right
- 答错 → result: wrong
- Mark 但答对 → result: right + conf: low（顺便记一下犹豫过）
- Mark 但答错 → result: wrong + conf: low

不要动 conf 和 my_note 列（除上面说的特殊情况），我会自己手动补关键题。

[贴入当前 yaml 块]
```

然后你只需要补 `conf`（中/低置信度的题）+ `my_note`（错题反思）。

---

### 阶段 4：复盘（填完结果立刻做）

**第 1 步**：打开 `_做题结果_NBME12.md`，往下滚到这几个区域：

- 📊 实时统计 → 看正确率
- 🎯 错题快速跳转 → 自动列出错题
- ⚠️ 伪掌握警报 → 低置信度答对的题

**第 2 步**：点击错题表格里的链接，跳到 `NBME12_breakdown.md` 对应 callout

**第 3 步**：在错题 callout 末尾追加【🤯 我为什么错】+【✏️ 复盘行动】区块（按你 v2 规则）

**第 4 步**：让 Claude Code 跑本套复盘 prompt（模板文件末尾有现成的）

---

### 阶段 5：跨套题分析（每做完 2-4 套）

打开 `NBME_汇总Dashboard.md`：
- 自动看到所有套题的对比表
- 跨套题伪掌握题清单
- 跨套题错题清单

做完 4+ 套后，运行 Dashboard 末尾的"综合复盘 Prompt"。

## ❓ 常见问题

### Q1: Dataview 表格不显示数据？

A: 检查：
1. 文件 frontmatter 里有 `tags: [做题结果]` 吗？
2. `qs:` 是有效 YAML 吗？（每个题前 `-` 缩进对齐）
3. Dataview 设置里 "Enable JavaScript Queries" 开了吗？

### Q2: 填了几题但统计还是 0？

A: Dataview 偶尔需要刷新。Ctrl+P → 输入 "Dataview: Force refresh all" → 回车。

### Q3: 我想加自定义字段（比如做题用时）？

A: 直接在 `qs:` 的每个题里加字段：

```yaml
  - qid: "001"
    result: right
    conf: high
    my_choice: A
    my_note: 
    time_seconds: 45     ← 新字段
```

然后改 `dataviewjs` 块加进表格显示。

### Q4: 填写时手酸怎么办？

A: 三个加速器：
1. **Find & Replace 批量**：先全填 `right`，再改特例
2. **双轨制**：让 Claude Code 帮填对错列
3. **极简模式**：只填 result，跳过 conf 和 my_note（损失 60% 复盘价值但比不填强）

### Q5: 是否需要每题都填 my_note？

A: 不需要。规则：
- ✅ 答对 + 高置信度 → 留空
- ✅ 答对 + 中置信度 → 可选
- 🔴 答对 + 低置信度（伪掌握）→ **建议填一句**（如"C/D 纠结"）
- ❌ 答错 → **必须填**（如"把 sens/spec 搞反"）
- 🏷️ 标记 → **必须填**（说明为什么不确定）

## 🎯 系统价值总结

| 你做的事 | 系统给你的回报 |
|---|---|
| 填一次 result | 自动计算正确率 |
| 填一次 conf | 自动识别伪掌握高危区 |
| 填一次 my_note | 给 Claude Code 复盘提供"思维路径" |
| 跨套题做 8 套 | Dashboard 自动汇总，无需手动维护 |
| 一句话错因 | 跨套题统计陷阱类型，找出反复中招模式 |

**单套填表时间**：10-30 分钟（视方法）
**单套复盘价值**：约 1-2 个高效自学小时（Claude Code 帮你识别盲区+定向复习）
**ROI**：填表 1 分钟 ≈ 复盘节省 5-10 分钟
